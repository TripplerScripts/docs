# Client
client side callback handling

## onPromise
define a promise that returns a value to the caller
```ts
onPromise<T extends (...args: any) => ReturnType<T>>(
  endpoint: string,
  Function: T
): boolean
```

---
### Parameters
* endpoint: The unique name of the promise endpoint
* Function: The function to run when the promise is triggered

---
### Return
* success: true if the promise was registered successfully

---
### Examples
```ts
import { onPromise } from '@trippler/tr_lib/client'

onPromise('tr_kit/client/test', (message: string) => {
  console.log(typeof message) /* Let's roll */
  return 'bingo bango bongo'
})
```

## triggerPromise
trigger a defined promise awaiting a value from the server
```ts
triggerPromise<T = unknown>(
  options?: [number, boolean] | number | boolean | string,
  endpoint?: string | any,
  ...parameters: any[]
): Promise<T | null>
```

---
### Parameters
* options?: [timeout in milliseconds, enable debug mode]
* endpoint: The unique name of the promise endpoint
* ...varargs?: The parameters to pass with the callback

---
### Return
* response: the return value of the server side promise

---
### Examples
```ts
import { triggerPromise } from '@trippler/tr_lib/client'

const message = await triggerPromise<{ message: string }>('tr_kit/server/test', `Let's roll`)

console.log(message) /* bingo bango bongo */
```
