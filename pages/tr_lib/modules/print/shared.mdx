# Print
a better way to print, so you can later look up and track where your codes at

### Conditions

* valid print type
* defined message field
* only the following aliases are allowed:
 * error = err
 * warning = warn
 * info = inf


<Callout type="warning">
* path and line are static values "as is" (except you know what you are doing) and also not required to be passes, they help for debugging and track the location of the actual print, except the ERROR type, when you ingore them, you will be switching automatically to returnal error that stops the execution of the rest of the code
</Callout>

 
<Tabs items={['Debug', 'Error', 'Warn', 'Info', 'Success', 'Log']}>
<Tabs.Tab>
## Debug

```lua
exports.tr_lib:print().debug(message, path, line)
```

### Parameters
* message: `string`
* path: `debug.getinfo(1, "Sl").short_src`
* line: `debug.getinfo(1, "Sl").currentline`

### Example

```lua filename="wrapper/print.lua"
function print(parameter)
    exports.tr_lib:print().debug(paramater)
end
```

```lua filename="shared/main.lua"
local retval = 'Hello Lenix'

print(retval, debug.getinfo(1, "Sl").short_src, debug.getinfo(1, "Sl").currentline)
```

<Callout type="info">
* its highly recommended to use debug.getinfo on the debug mode
</Callout>
</Tabs.Tab>
